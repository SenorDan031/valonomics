# Developer Log Formatting Rules

All contributors must follow this standardized format when writing dev reports.

These rules ensure clarity, consistency, and traceability across the VALONOMICS project.

---

## General Rules set 2 

- Dev logs must be written only for meaningful development milestones.
  
- Avoid logging minor changes or micro-fixes.
  
- Use formal technical language.
  
- Each contributor must sign their work with their GitHub profile.
  
- New daily entries must always be placed at the top of the file.
  
- The structure must remain unchanged unless officially updated by the project maintainer.

---

## Single Developer Entry Format

```md
Author: [NAME](https://github.com/username)

## Work Completed

- Bullet points describing completed tasks
- Architecture changes
- Feature implementations
- Refactors or major fixes

-----

(E O L)

```

## Multiple Developer Entry Format

```md

   Author: [NAME](https://github.com/username)

     ## Work Completed

        - Individual contributions listed here

   -----

  (E O L)

  Author: [NAME](https://github.com/username)

    ## Work Completed

       - Individual contributions listed here

  -----

  (E O L)
